# Security Policy

## Reporting a Vulnerability

We take the security of RepoLens seriously. If you believe you've found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to the maintainers.

## Scope

Security vulnerabilities in RepoLens itself (the CLI tool) are in scope. This includes:

- Code execution vulnerabilities
- Path traversal issues
- Sensitive data exposure
- Dependency vulnerabilities

## What to Expect

We will acknowledge receipt of your vulnerability report within 72 hours and send a more detailed response within 5 business days.

## Preferred Languages

We prefer all communications to be in English.
